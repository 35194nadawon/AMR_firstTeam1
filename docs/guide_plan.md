## 🟦 [안내팀 R2] 하방 카메라 추가(URDF) 및 YOLOv8 점자 블록 인식 비전 구현

Markdown

```
### 🎯 개발 목표
로봇이 바닥에 설치된 황색 점자 블록(선형/점형 블록)을 따라 정밀하게 주행할 수 있도록, 로봇 모델(URDF)에 하방 카메라를 추가하고 YOLOv8을 활용해 점자 블록을 실시간 객체 탐지(Object Detection)하는 비전 노드를 구현합니다.

### 🛠️ 세부 작업 및 구현 체크리스트
- [ ] **URDF 모델 수정 및 하방 카메라(Downward Camera) 추가**
  - 숨는팀의 전방 카메라와의 센서 충돌 및 간섭을 방지하기 위해 `storagy` 로봇 모델 URDF(SDF) 파일 수정
  - 바닥면을 수직 또는 비스듬히 바라보는 가상의 카메라 링크 및 Gazebo 센서 플러그인 추가
- [ ] **YOLOv8 기반 황색 점자 블록 데이터셋 확보 및 학습**
  - Gazebo 시뮬레이션 환경 내 바닥 텍스처(황색 선형 블록, 점형 블록)에 맞는 맞춤형 YOLO 데이터셋 구성 또는 오픈소스 데이터 활용 학습 수행
  - `yolo_detector.py`를 확장하거나 신규 노드를 생성하여 하방 카메라 이미지 이미지 스트림 처리
- [ ] **점자 블록 중심선 오차 및 특징점 계산**
  - 검출된 선형 블록의 바운딩 박스 중심점과 로봇의 중심(정중앙 축) 사이의 횡방향 오차($e_y$) 및 각도 오차($e_\theta$) 계산
  - 계산된 오차 배열 데이터를 오차 보정 노드(R3)로 전송할 커스텀 메시지 또는 정형 토픽 설계

### 📌 데이터 인터페이스 & 인계 규칙
- **구독 토픽 (Input)**: `/camera/downward/image_raw` (`sensor_msgs/msg/Image`)
- **발행 토픽 (Output)**: `/guide/yellow_line_error` (`geometry_msgs/msg/Pose2D` 또는 커스텀 정형 데이터, 오차 값 전송용)
- **협의 사항**: 숨는팀 R3(사람 감지 비전)와의 GPU 리소스 분배 및 가제보 월드 텍스처(점자블록 배치) 동기화 필요
```
