### 🎯 개발 목표
로봇이 목적지 테이블에 도달했을 때의 정밀 정지 제어를 수행하고, 손님에게 점자 메뉴판 제공 안내(O디오/OLED 싱크)를 수행한 뒤, 최종 서비스 완료 시 제어권을 완벽히 해제하고 숨는팀에게 복귀 명령을 토스하는 전반적인 시나리오 엔딩 시퀀스를 구현합니다.

### 🛠️ 세부 작업 및 구현 체크리스트
- [ ] **목적지 도달 정밀 정지 및 상위 제어 이벤트 연동**
  - Nav2 Goal 달성 상태(Success)를 트래킹하여 로봇 정지 처리 및 목적지 매치 검증
- [ ] **메뉴판 제공 및 HRI 싱크 (OLED/오디오)**
  - 목적지 도착 시 `Emotion.srv` 또는 관련 토픽을 통해 로봇 OLED 디스플레이에 "메뉴판 수령 안내" 그래픽 송출
  - 시각장애인용 안내 오디오(시뮬레이션 환경 상 로그 출력 또는 가상 오디오 노드) 싱크 일치
- [ ] **제어권 양보 메커니즘 보장 (숨는팀 시작 블로커 해제 핵심)**
  - 손님이 메뉴판을 수령하거나 서비스 완료 버튼이 트리거되면, 기존 안내팀이 점유하던 모든 주행 루틴(`wander_nav_bt_node`) 비활성화 처리
  - 내부 `robot_tools._stop_all_motion()`을 호출하여 내비게이션 Goal 취소 및 `/cmd_vel` 발행을 완전히 중단하여 숨는팀 도킹 노드가 제어권을 넘겨받을 수 있도록 공백 상태(Arbitration) 보장
- [ ] **임무 완료(MISSION DONE) 최종 신호 송신**
  - 제어권 해제 직후 숨는팀 FSM이 복귀(RETURN) 모드로 즉시 전이될 수 있도록 `mission_done` 플래그 토픽 발행

### 📌 데이터 인터페이스 & 인계 규칙 (숨는팀과의 인계 계약)
- **발행 토픽 (Output)**: `/hide/mission_done` (`std_msgs/msg/Bool`, 단일 펄스 True 발행)
- **보장 조건 (Crucial)**: `mission_done` 발행 직전 안내팀의 모든 `/cmd_vel` 발행 중단 및 Nav2 액션 클라이언트 Close 상태 확보 필수
- **참고 구현**: `src/storagy/scripts/storagy_llm/robot_tools.py` 내
