## [안내팀 R1] 음성 인터페이스(STT) 및 LLM 인텐트 백엔드 구축

Markdown

```
### 🎯 개발 목표
사용자(시각장애인 손님)의 스마트폰 음성 입력을 받아 텍스트로 변환(STT)하고, 기존 `storagy_llm` 에이전트와 연동하여 손님의 의도(빈자리 안내, 카운터 행 등)를 파악한 뒤, 매장 테이블 상태 데이터에 맞춰 목적지 좌표를 매핑하는 상위 제어 시스템을 구축합니다. 숨는팀의 시작 트리거를 발행하는 핵심 관문입니다.

### 🛠️ 세부 작업 및 구현 체크리스트
- [ ] **웹/스마트폰 마이크 기반 STT 파이프라인 추가**
  - 현재 텍스트/CLI 기반인 입력을 브라우저 Web Audio API 또는 스마트폰 마이크 입력 -> STT(OpenAI Whisper, Google STT 등) -> LLM 파이프라인으로 확장
- [ ] **LLM 인텐트 파싱 및 빈자리 테이블 상태 매핑**
  - "조용한 빈자리로 가줘", "카운터로 안내해줘" 등의 음성 맥락을 분석하는 프롬프트 최적화
  - 가상의 매장 테이블 데이터베이스(FastAPI 내 메모리 컨텍스트 등)와 연동하여 현재 '빈자리' 상태인 최적의 테이블 좌표(`table_3` 등) 매칭 로직 구현
- [ ] **숨는팀 기상(WAKE) 트리거 발행 (의존성 필수)**
  - 음성 명령이 정상 확인되면, 구석에 숨어있는 숨는팀 로봇을 깨우기 위한 트리거 발행
  - `takeover_start` 신호 전송 로직 구현
- [ ] **대시보드 시크릿 도슨트 모드 UI 전환 지원**
  - 숨는팀이 발행하는 `/hide/state` 토픽을 구독하여, 로봇 상태가 `WAKE` 또는 `GUIDE`로 전환될 때 웹 대시보드 UI가 '안내/도슨트 모드'로 동적 전환되도록 프론트엔드/SSE(Server-Sent Events) 스트림 연동

### 📌 데이터 인터페이스 & 인계 규칙
- **발행 토픽 (Output)**: `/hide/takeover_start` (`std_msgs/msg/Bool`, 단일 펄스 True 발행)
- **구독 토픽 (Input)**: `/hide/state` (`std_msgs/msg/String`, 대시보드 UI 상태 전환용)
- **참고 파일**: `src/storagy/scripts/storagy_llm/agent_service.py`
```
