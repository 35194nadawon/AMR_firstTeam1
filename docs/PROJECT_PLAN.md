# Project Plan

## 프로젝트명

Toy Guide - 안내견을 위한 시크릿 보조 로봇

## 한 줄 목표

평상시에는 인형처럼 숨어 있다가, 시각장애인 사용자가 입장하면 안내견의 실내 안내 임무를 교대받아 안전하게 목적지까지 안내하고 다시 대기석으로 복귀하는 ROS2 자율주행 로봇을 만든다.

## 핵심 시나리오

### 1단계: 위장 및 대기

- 로봇은 카페/음식점 구석의 AprilTag 대기석에 정밀 도킹한다.
- OLED/LED는 대기 상태로 전환한다.
- 사람 이동이 많은 구역은 Costmap에 반영하여 대기 위치를 안정적으로 유지한다.

### 2단계: 무전 수신 및 임무 교대

- 스마트폰 앱, 비콘, 웹 대시보드, 또는 LLM 명령으로 안내 시작 이벤트를 받는다.
- 로봇은 대기 모드에서 깨어나 입구 또는 사용자 위치로 이동한다.
- 웹 대시보드는 매장 관리 화면에서 시크릿 도슨트 모드로 전환한다.

### 3단계: 점자 블록 인식 및 안전 안내

- 하방 카메라로 점자 블록을 인식한다.
- Nav2 경로 추종과 비전 기반 조향 보정을 함께 사용한다.
- IMU 변화로 문턱, 요철, 충격을 감지하고 속도 또는 모터 출력을 보정한다.

### 4단계: 돌발 상황 회피 및 복귀

- 사람 또는 장애물을 감지하면 즉시 Freeze한다.
- OLED/음성/웹으로 "안내 중입니다. 길을 양보해 주세요" 상태를 알린다.
- 위험이 사라지면 Resume한다.
- 목적지 도착 후 대기석으로 복귀하고 AprilTag로 정밀 도킹한다.

## 역할 분담

| 역할 | 담당 범위 | 주요 산출물 |
| --- | --- | --- |
| Integration / PM | 전체 시나리오, launch 통합, 데모 플로우 | 통합 launch, 데모 체크리스트 |
| Navigation | Nav2, Freeze/Resume, Costmap Filters, 복귀 주행 | `toy_guide_navigation` |
| Vision | YOLOv8 사람/점자블록 감지, AprilTag 도킹 | `toy_guide_vision` |
| Mission / LLM | 임무 상태 머신, 명령 파싱, 메시지 생성 | `toy_guide_mission` |
| MCU / Control | IMU, 모터 보정, OLED/LED 제어 | `toy_guide_mcu_bridge` |
| Web Dashboard | 상태 표시, 경고, 시나리오 연출 | `toy_guide_dashboard_bridge` |

## 마일스톤

### M1: 프로젝트 기반 구축

- 기존 Storagy 실행 절차 확인
- 패키지 구조 확정
- ROS 인터페이스 합의
- GitHub 이슈/PR 템플릿 사용 시작

### M2: 감지와 정지

- YOLO 사람 감지 노드
- LiDAR 장애물 감지 연동
- Freeze/Resume 상태 머신
- `/cmd_vel` 안전 제어 방식 결정

### M3: 안내 주행

- 목적지 waypoint 등록
- 점자 블록 인식
- Nav2 goal 발행
- 점자 블록 중심 오차 기반 조향 보정

### M4: 은폐 복귀

- 사람 시야각 120도 위험 구역 생성
- Costmap Filter 또는 동적 장애물 반영
- AprilTag 대기석 정밀 도킹

### M5: 통합 데모

- 웹 대시보드 연동
- LLM 무전 문구 출력
- OLED/LED/음성 출력
- 전체 시나리오 리허설 및 영상 촬영

## 데모 성공 기준

- 기존 Storagy bringup이 깨지지 않는다.
- 사람 감지 시 0.5초 이내 Freeze한다.
- 위험 해소 후 Resume 명령으로 안내를 계속한다.
- 목적지 도착 상태가 ROS 토픽과 웹 대시보드에 동시에 표시된다.
- 대기석 복귀 후 AprilTag 기준으로 정렬 상태를 확인할 수 있다.
